Come sono stati pensati e sviluppati i java beans della directory corrente?

I java beans sono file di codice scritto in java che ci permette di implementare nel linguaggio del sito web(JAVA) tutti gli oggetti in esso
: Articoli,Bonus,Utenti,ecc..

Un Java Bean è un componente software che ci permette di gestire e rappresentare i dati e di mantenerli durante la logica dell'applicazione.

Come è formato un JavaBean?
 
 -Nel nostro caso è formato con il nome della Classe che rappresenta l'oggetto + Bean che sta ad indicare che il file è un JavaBean.
 
 -Oltre ad un costruttore che setta l'oggetto come un elemento nullo , quindi impostando valori dei suoi attributi che non vadano a rappresentare 
  un dato effettivo , abbiamo i metodi getter e setter per ogni attributo. Banalmente ci serviranno a reperire le informazioni dagli oggetti o ad
   aggiornarle.
   
  -L'interfaccia Serializable in Java è utilizzata per abilitare la serializzazione degli oggetti, cioè la conversione di un oggetto in un formato
    che può essere facilmente salvato su disco (persistenza), inviato attraverso una rete o memorizzato in una sessione web.

Per salvare le date utilizziamo la classe LocalDate contenuta in java.time.LocalDate in modo tale da mantenere anche il formato utile per sql
LocalDate.of(aaaa,mm,gg) e LocalDate.now() sono due metodi utili

ESEMPIO CONVERSIONE DEL TIPO LOCALDATE JAVA AL TIPO DATE DI SQL    

import java.time.LocalDate;
import java.sql.Date;

public class LocalDateToSqlDateExample {
    public static void main(String[] args) {
        // Creare un oggetto LocalDate
        LocalDate localDate = LocalDate.now();
        System.out.println("LocalDate: " + localDate);

        // Convertire LocalDate in java.sql.Date
        Date sqlDate = Date.valueOf(localDate);
        System.out.println("java.sql.Date: " + sqlDate);
    }
}
